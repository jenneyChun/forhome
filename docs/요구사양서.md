# ForHome Requirements

## Goal

ForHome records family chores, fatigue, badges, and family messages on a PC-hosted local web server. Desktop and mobile browsers on the same network should use the same PostgreSQL-backed data.

## Requirements

1. Store application data in PostgreSQL tables, not JSON state files.
2. Separate source code from runtime user data and generated exports.
3. Organize folders by role: `code`, `server`, `data`, `log`, `tests`, and `docs`.
4. Keep future guide documents in `docs`.
5. Keep Codex implementation session notes in `docs/session`, ordered by creation date.
6. Add verification test cases for implemented behavior.
7. Reflect completed work to the GitHub remote repository.

## Functional Scope

- Family member login
- Admin settings
- Chore completion records
- Weekly fatigue and XP summary
- Badge history
- Family messages
- Calendar activity view
- Kakao daily summary export and send script

## PostgreSQL Tables

Main tables:

- `members`
- `accounts`
- `chores`
- `history`
- `messages`
- `badge_history`
- `member_badges`
- `settings`
- `app_version`

Files under `data/exports` are generated integration artifacts. They are not the primary application data store.

## Verification

- PowerShell scripts must parse successfully.
- Server code must not depend on `app_state.json`.
- PostgreSQL schema must define the major tables.
- With PostgreSQL installed, schema initialization and seeded state reads must pass.
- With Node/npm/Playwright installed, desktop and mobile browser flows must pass.
