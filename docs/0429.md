## Firebase를 계속 쓰는 방법

Firebase를 계속 쓰면서 여러 사용자가 같은 정보를 공유하려면 Firebase에서 제공하는 DB를 쓰면 됩니다. 이 앱에는 보통 다음 두 가지 중 하나가 적합합니다.

### Firestore

Firestore는 문서 기반 DB입니다. 가족 앱의 전체 상태를 하나의 문서에 저장하거나, 가족 구성원/집안일/기록/메시지를 컬렉션으로 나눠 저장할 수 있습니다.

가능한 단순 구조:

```text
families/{familyId}
families/{familyId}/members/{memberId}
families/{familyId}/chores/{choreId}
families/{familyId}/history/{historyId}
families/{familyId}/messages/{messageId}
```

장점:

- 모바일/PC/다른 브라우저에서 같은 데이터를 실시간으로 공유 가능
- `onSnapshot`으로 변경사항을 자동 반영 가능
- Firebase Hosting과 같이 쓰면 PC 서버 없이 접속 가능
- 사용자 인증을 붙이면 가족별 접근 권한을 제어 가능

주의할 점:

- 브라우저 코드에 Firebase 설정값은 들어가도 되지만, 보안은 Firestore Security Rules로 막아야 함
- 모든 데이터를 한 문서에 몰아넣으면 문서 크기와 동시 수정 충돌 문제가 생길 수 있음
- 가족별 `familyId`와 로그인/권한 설계를 정해야 함

추천 방향:

1. 앱은 Firebase Hosting 또는 GitHub Pages에 올립니다.
2. 데이터는 Firestore에 저장합니다.
3. 로그인은 Firebase Authentication을 쓰거나, 최소한 가족 전용 공유 코드/비밀번호를 둡니다.
4. `history`, `messages`처럼 계속 늘어나는 데이터는 별도 컬렉션으로 분리합니다.

### Realtime Database

Realtime Database도 가능하지만, 이 앱처럼 기록/메시지/집안일/가족 구성원을 구조적으로 다루려면 Firestore가 더 관리하기 쉽습니다. 단순한 실시간 동기화만 중요하면 Realtime Database도 선택할 수 있습니다.

## GitHub에 요일별로 데이터를 저장하는 방법

GitHub 저장 방식도 가능합니다. 다만 GitHub는 앱의 실시간 DB라기보다는 백업/보고서 저장소로 쓰는 편이 더 적합합니다.

가능한 저장 경로 예시:

```text
data/daily/2026-06-03.json
reports/weekly/2026-06-01.md
reports/monthly/2026-06.md
```

요일별 백업 방식:

```text
data/by-weekday/monday.json
data/by-weekday/tuesday.json
data/by-weekday/wednesday.json
data/by-weekday/thursday.json
data/by-weekday/friday.json
data/by-weekday/saturday.json
data/by-weekday/sunday.json
```

또는 날짜 기준으로 저장하는 방식이 더 안전합니다.

```text
data/daily/YYYY-MM-DD.json
```

장점:

- GitHub에 변경 이력이 남음
- 주간/월간 보고서와 백업 파일을 관리하기 쉬움
- 앱 데이터 복구용으로 좋음

주의할 점:

- 브라우저 앱 안에 GitHub Personal Access Token을 직접 넣으면 토큰이 노출될 수 있음
- 여러 사용자가 동시에 저장하면 충돌이 날 수 있음
- 실시간 공유 DB로 쓰기에는 Firebase보다 불편함

추천 방향은 Firebase를 기본 공유 DB로 쓰고, GitHub는 일일/주간 백업과 보고서 저장에 사용하는 방식입니다.

## 추천 구조

현재 목표가 “모바일과 다른 사용자가 같은 데이터를 공유하는 앱”이라면 아래 구조가 가장 자연스럽습니다.

```text
앱 배포: Firebase Hosting 또는 GitHub Pages
공유 DB: Firebase Firestore
백업/보고서: GitHub 저장소에 날짜별 JSON/Markdown 저장
카카오 알림: 일일 요약 링크는 배포된 앱 주소로 연결
```

이렇게 하면 PC를 서버로 계속 켜둘 필요 없이, 가족 구성원이 각자 모바일 브라우저에서 같은 앱에 접속하고 같은 Firestore 데이터를 볼 수 있습니다.
